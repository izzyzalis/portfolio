The live app (aqua-analyzer.streamlit.app) is a bare Streamlit UI — the interesting part is the pipeline behind it: pull text out of a PDF water-quality report, hand it to GPT, and serve the answer back conversationally. Here's the actual code, lightly cleaned up. 

# pulls the raw text out of the monthly water quality PDF
import fitz  # PyMuPDF

with fitz.open(pdf_path) as doc:
    extracted_text = "".join(page.get_text() for page in doc)
# sends the report text + the resident's question to GPT
from openai import OpenAI
client = OpenAI(api_key=os.environ["OPENAI_API_KEY"])

prompt = (
    f"This is a water quality report from San Jose.\n"
    f"Report content:\n{extracted_text}\n\n"
    f"Now answer this for someone who isn't a water expert:\n{user_question}"
)
response = client.chat.completions.create(
    model="gpt-3.5-turbo",
    messages=[
        {"role": "system", "content": "You explain San Jose water conditions to residents."},
        {"role": "user", "content": prompt}
    ]
)
answer = response.choices[0].message.content.strip()
import streamlit as st

st.title("💧 Valley Water Mineral Guide")
user_question = st.chat_input("Ask a question about the water report or minerals")

if user_question:
    with st.spinner("Thinking..."):
        answer = ask_gpt(user_question)
        st.markdown(f"**💡 Answer:** {answer}")

Note: the key is loaded from an environment variable here, never hardcoded — basic hygiene for anything you'd actually ship.
